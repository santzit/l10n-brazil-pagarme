# Copilot Instructions for l10n-brazil-pagarme

## Repository Overview

This is an **ODOO 16.0 OCA (Odoo Community Association) repository** for the Pagarme payment provider with Brazilian localization. The repository follows OCA standards and best practices for Odoo module development.

## Repository Structure

This repository contains a single Odoo module following OCA directory standards:


### 8. Development Workflow

#### New Feature Development
1. **Create feature branch**: `git checkout -b feature/payment-improvement`
2. **Follow OCA structure**: Add models, views, tests following existing patterns
3. **Run pre-commit**: Ensure code quality before committing
4. **Add tests**: Create comprehensive tests for new functionality
5. **Update manifest**: Increment version following OCA guidelines
6. **Submit PR**: Follow OCA contribution guidelines

#### Bug Fixes
1. **Identify issue**: Check existing tests and module behavior
2. **Create test**: Add test that reproduces the bug
3. **Fix code**: Make minimal changes to fix the issue
4. **Verify fix**: Ensure test passes and no regressions
5. **Update version**: Increment patch version

### 9. OCA Specific Guidelines

#### Dependencies
- **Odoo dependencies**: List in `depends` field of manifest
- **Python dependencies**: Add to requirements.txt if needed
- **External dependencies**: Document in `external_dependencies` manifest field

#### Translations
- Use Odoo's translation system: `_("Translatable string")`
- Generate .pot files automatically with OCA tools
- Follow OCA translation contribution guidelines

#### Documentation
- Keep module description brief in manifest
- Add comprehensive docstrings to methods
- Update README.md with OCA template (auto-generated)

### 10. Best Practices

#### Code Organization
- **One model per file** in models/ directory
- **Logical grouping** of related functionality
- **Proper inheritance** using `_inherit` or `_name`

#### Error Handling
```python
from odoo.exceptions import UserError, ValidationError
```
